  ]

5
Admin — Review & Approve
Admin reviews the full profile (bikes + equipment + certs) then approves with final prices per level.
5-1
POST ADMIN
Admin Login
/api/admin/login
Saves: {{admin_token}}

Body

{
  "login":    "admin@dabapp.com",
  "password": "YourAdminPassword"
}
5-2
GET ADMIN
Get Pending Trainers
/api/admin/trainers?status=pending
Saves: {{trainer_id}} from first result.
Returns full profile including bikes, equipment, certificate files, and requested prices.
5-3
GET ADMIN
Get Trainer Detail
/api/admin/trainers/{{trainer_id}}
Review bikes, equipment, certification files and requested prices before approving.
5-4
POST ADMIN
Approve Trainer + Set Final Prices
/api/admin/trainers/{{trainer_id}}/approve-levels
After this: Trainer status = approved. Phase 6 can now run.

Body

{
  "levels": [
    { "level_id": 1, "approved_price": 150 },
    { "level_id": 2, "approved_price": 220 },
    { "level_id": 3, "approved_price": 350 }
  ],
  "approve_trainer": true
}
approved_price is the price clients will see and pay. Trainer's requested_price is for admin reference only.

To reject instead

// POST /api/admin/trainers/{{trainer_id}}/reject
{
  "reason": "Required certifications could not be verified."
}

8
Client — Book Travel Course (Course B)
Preview the total price including travel cost first, then book. No location needed — trainer comes to client.
8-1
GET
Calculate Travel Price
/api/trainers/{{trainer_id}}/travel-price
Public. Run this before booking so the client sees the exact total. Only works on courses with can_travel=true.

Query params

course_id = {{course_b_id}}
km        = 20           // distance from trainer to client location

Expected response

{
  "data": {
    "km":                20,
    "price_per_km":      5,
    "travel_cost":       100,    // 20 × 5
    "session_price":     200,    // course original_price
    "total_with_travel": 300     // 200 + 100 ← client pays this
  }
}
Formula: total = original_price + (km × price_per_km)

9-1
POST
Trainer: Start Session
/api/trainer/sessions/{{booking_a_id}}/start
Use {{trainer_token}}. No body.
Returns 403 if payment_status != paid — client must pay first.

confirmed → in_progress

9-2
POST
Trainer: Complete Session
/api/trainer/sessions/{{booking_a_id}}/complete
Use {{trainer_token}}. No body. Client is notified to confirm.

in_progress → awaiting_confirmation

9-3
POST CLIENT

/api/trainer/bookings/{{booking_a_id}}/confirm-completion
Switch back to {{token}}. No body.

awaiting_confirmation → completed. can_review becomes true.

/api/trainer/bookings/{{booking_a_id}}/review
Only once per booking. Only after status = completed.

Trainer role · Folder: Trainer / Login + Registration

Start here. The trainer logs in and submits the registration form. Status will be pending until admin approves.

1
POST
Trainer Login
/api/login
Saves: {{trainer_token}} automatically via test script.

Request body

// POST /api/login
{
  "login":    "trainer@dabapp.com",
  "password": "YourTrainerPassword"
}

Success response 200

{
  "success": true,
  "token":   "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."
}

POST
Upload Profile Photo (optional)
/api/trainer/upload-photo
Use form-data. Key: photo, Type: File. Returns the storage path to use in the register body.

Success response 200

{ "path": "trainers/photos/abc123.jpg" }

Trainer — Add Bikes to Garage
Trainer role · Folder: Trainer / My Garage [STEP 1 – Add Bikes]

Before selecting training bikes, the trainer must first add motorcycles to their personal garage. Run steps 1→2→3 to get valid IDs, then POST the bike.

1
GET PUBLIC
Get Motorcycle Brands
/api/motorcycle/brands
No auth required. Check the console log for brand IDs.

Example response

[
  { "id": 7,  "name": "BMW" },
  { "id": 12, "name": "Honda" },
  { "id": 3,  "name": "Yamaha" }
]

📋 Note the id of the brand you want. Use it in step 2.

2
GET PUBLIC
Get Models by Brand
/api/motorcycle/models/{brandId}
Replace 7 in the URL with your brand ID from step 1.

Example: GET /api/motorcycle/models/7 (BMW)

[
  { "id": 1, "name": "GS 1250" },
  { "id": 2, "name": "R 1250 RT" }
]

📋 Note the model id. Use it in step 3.

3
GET PUBLIC
Get Years by Model
/api/motorcycle/years/{modelId}
Replace 1 in the URL with your model ID from step 2.

Example: GET /api/motorcycle/years/1

[
  { "id": 3, "year": 2025 },
  { "id": 4, "year": 2024 }
]

📋 Note the year id. Use it in step 4.

4
POST AUTH
Add Bike to Garage
/api/my-garage
Saves: {{garage_id}} automatically.

Request body

{
  "brand_id":   7,   // from step 1
  "model_id":   1,   // from step 2 (must belong to brand)
  "year_id":    3,   // from step 3 (must belong to model)
  "title":      "My BMW GS",    // optional nickname
  "description": "Daily touring bike"
}

Success response 201

{
  "success": true,
  "data": {
    "id":    3,              // ← this is your {{garage_id}}
    "brand": { "name": "BMW" },
    "model": { "name": "GS 1250" },
    "year":  { "year": 2025 },
    "plate_number":              null,
    "insurance_expiry":          null,
    "insurance_covers_training":  null
  }
}
5
PATCH AUTH
Add Training Info (plate + insurance) ← REQUIRED
/api/my-garage/{{garage_id}}/training-info
Required before step 6! Without all 3 fields, the bike shows is_training_ready: false and cannot be used for training sessions.

Request body

{
  "plate_number":             "ABC 1234",
  "insurance_expiry":         "2027-01-01",
  "insurance_covers_training": true
}

Success response 200

{
  "success": true,
  "data": {
    "id":                         3,
    "plate_number":              "ABC 1234",
    "insurance_expiry":          "2027-01-01",
    "insurance_covers_training":  true,
    "is_training_ready":          true   // ← must be true
  }
}
6
GET AUTH
List My Garage
/api/my-garage

Returns all bikes owned by the authenticated user, with brand/model/year and training info.

7
GET AUTH
Get a Single Garage Bike
/api/my-garage/{{garage_id}}

Detail view of one bike. 404 if it doesn't belong to you.

8
GET AUTH
Get Default Garage Bike
/api/my-garage/default

Returns the bike marked as default (used as a fallback elsewhere in the app).

9
POST AUTH
Set Bike as Default
/api/my-garage/{{garage_id}}/set-default

Marks this bike as the default and unmarks any previous default. No body needed.

10
GET AUTH
Check Training Completeness
/api/my-garage/training-completeness

Quick summary across all your bikes of which ones are is_training_ready and which are missing plate/insurance fields.

11
PUT AUTH
Update Garage Bike
/api/my-garage/{{garage_id}}

Request body (any subset)

{
  "title":       "My BMW GS (Track Setup)",
  "description": "Updated for circuit use"
}
Also accepts PATCH on the same URL for partial updates.
12
DELETE AUTH
Delete Garage Bike
/api/my-garage/{{garage_id}}
If the bike is currently selected as a training bike, remove it from Training Bikes (Scenario 3) first.

Trainer — Select Training Bikes from Garage
Trainer role · Folder: Trainer / Training Bikes [STEP 2]

Select which garage bikes will be used for training sessions. Requires the bike to have training info filled (previous section step 5).

1
POST AUTH
Select Bike as Training Bike
/api/trainer/training-bikes

Request body

{
  "garage_id":  3,      // {{garage_id}} from garage step
  "is_primary": true   // first bike should be primary
}

Success response 201

{
  "success": true,
  "message": "Training bike added successfully",
  "data": {
    "trainer_id": 4,
    "garage_id":  3,
    "is_primary": true
  }
}

Error responses

// 409 — bike already selected
{ "success": false, "message": "This bike is already added as a training bike" }

// 403 — bike not in your garage
{ "success": false, "message": "Bike not found in your garage" }
2
GET AUTH
Verify Training Bikes List
/api/trainer/training-bikes

Expected response

{
  "success": true,
  "data": [
    {
      "id":               1,
      "garage_id":       3,
      "is_primary":      true,
      "is_training_ready": true,   // ← confirms plate+insurance filled
      "garage": {
        "plate_number": "ABC 1234",
        "brand":  { "name": "BMW" },
        "model":  { "name": "GS 1250" },
        "year":   { "year": 2025 }
      }
    }
  ]
}
3
PATCH AUTH
Set Training Bike as Primary
/api/trainer/training-bikes/{{garage_id}}/set-primary

Marks this garage bike as the primary training bike and clears the flag on all others. No body needed.

4
DELETE AUTH
Remove Training Bike
/api/trainer/training-bikes/{{garage_id}}
Only removes it from the training-bike list — the bike stays in My Garage.

Admin — Verify Profile & Approve Levels
Admin role · Folder: Admin / Trainer Applications

Admin can now see the trainer's complete profile: bikes, equipment, certifications. Admin approves specific training levels with their prices.

1
POST
Admin Login
/api/admin/login
Saves: {{admin_token}} automatically.
{
  "login":    "admin@dabapp.com",
  "password": "YourAdminPassword"
}
2
GET
Get Pending Trainers
/api/admin/trainers?status=pending
Saves: {{trainer_id}} from the first pending trainer.

Returns a paginated list of all trainers with status=pending. Includes their bikes and equipment.

3
GET
View Full Trainer Profile (admin)
/api/admin/trainers/{{trainer_id}}

Admin views full profile: certifications, bikes, equipment, requested level prices. Use this to decide which levels to approve.

4
POST
Approve Levels with Prices
/api/admin/trainers/{{trainer_id}}/approve-levels
approve_trainer: true sets the trainer status to approved. Without it, the levels are saved but trainer stays pending.

Approve all 3 levels

{
  "levels": [
    { "level_id": 1, "approved_price": 150 },
    { "level_id": 2, "approved_price": 220 },
    { "level_id": 3, "approved_price": 350 }
  ],
  "approve_trainer": true
}

Approve beginner level only

{
  "levels": [
    { "level_id": 1, "approved_price": 150, "notes": "Certifications verified" }
  ],
  "approve_trainer": true
}

Reject a specific level

// POST /api/admin/trainers/{{trainer_id}}/reject-level
{
  "level_id": 3,
  "notes": "Advanced certifications could not be verified."
}

9
POST AUTH
Create Course with Travel Option
/api/trainer/courses
Instead of a fixed location_id, the trainer travels to the client. Requires can_travel=true and price_per_km.
{
  "title":             "Mobile Coaching — I Come to You",
  "hours_per_session": 2,
  "total_sessions":    3,
  "original_price":    200,
  "can_travel":         true,
  "price_per_km":       5
}
Tip: save this course's id as {{travel_course_id}} (separate from {{course_id}}) so step 10 below always tests the right course.
10
GET PUBLIC
Calculate Travel Price from Number of KM
/api/trainers/{{trainer_id}}/travel-price
This is how the client (or trainer, to preview) gets the total price for a can_travel=true course: total_price = original_price + (price_per_km × km). Works only on courses created with can_travel=true and price_per_km set — like step 9 above.
// Query params
course_id = {{travel_course_id}}   // the course from step 9 (price_per_km = 5)
km        = 12

Example response 200

{
  "success": true,
  "data": {
    "km":           12,
    "price_per_km": 5,
    "travel_fee":   60,    // 12 × 5
    "base_price":   200,   // course original_price
    "total_price":  260   // base_price + travel_fee
  }
}
403 Travel price not available for this course if you pass a course_id that has can_travel=false (e.g. the fixed-location course from step 1).

Client — Book a Session
Client role · Folder: Client / Availability & Booking

Client checks availability then books a session. A payment URL is returned to simulate payment.

1
GET AUTH
Check Trainer Availability
/api/trainers/{{trainer_id}}/availability
Pre-request script auto-fills from_date and to_date for this week.

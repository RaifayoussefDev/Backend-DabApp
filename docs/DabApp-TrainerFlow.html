POST AUTH
Register Trainer Profile
/api/trainer/register
Auto-saves: {{trainer_id}} from response data.trainer_id
{
  "bio":         "Professional motorcycle instructor with 8+ years of track experience.",
  "bio_ar":      "مدرب دراجات نارية محترف مع أكثر من 8 سنوات من خبرة الحلبة.",

POST AUTH
Add Bike to Garage
/api/my-garage
Auto-saves: {{garage_id}}
{
  "brand_id": 1,   // {{brand_id}}
  "model_id": 1,   // {{model_id}}
  "year_id":  1,   // {{year_id}}

PATCH AUTH
Add Training Info (plate + insurance)
/api/my-garage/{{garage_id}}/training-info
Required! Without these 3 fields the bike cannot be used for training (is_training_ready: false).

  "insurance_expiry":         "2027-12-31",
  "insurance_covers_training": true
}

GET PUBLIC
Get Equipment Catalog
/api/equipment-types
Logs all IDs to console. Admin must run EquipmentTypeSeeder first.

POST AUTH
Select Equipment from Catalog
/api/trainer/equipment
Names/icons are copied from catalog automatically. Duplicate IDs are skipped.
{
  "equipment_type_ids": [1, 2, 3, 4, 5, 6]
}

POST AUTH
Select Training Bike
/api/trainer/training-bikes
{
  "garage_id":  1,    // {{garage_id}}
  "is_primary": true
}
403 if bike has is_training_ready: false — complete step 7 first.

1
POST ADMIN
Admin Login

{
  "login":    "admin@dabapp.com",

2
GET ADMIN
Get Pending Trainers
/api/admin/trainers?status=pending
Auto-saves: {{trainer_id}} from first result

Find the trainer registered in Phase 1. Includes their bike, equipment and requested level prices.

3
POST ADMIN
Approve Trainer + Levels
/api/admin/trainers/{{trainer_id}}/approve-levels
approve_trainer: true sets status to approved in the same call.
{
  "levels": [
    { "level_id": 1, "approved_price": 150 },
    { "level_id": 2, "approved_price": 200 }
  ],
  "approve_trainer": true
}
After this the trainer status becomes approved. Phase 3 can now run.

Client — Book Course B (Travel)
Folder: Phase 5 – Book Travel Course · total_price = original_price + (km × price_per_km)
1
GET PUBLIC
Calculate Travel Price
/api/trainers/{{trainer_id}}/travel-price
Run this before booking so the client sees the total price including travel fee. Only works on courses with can_travel=true.
// Query params
course_id = {{course_b_id}}
km        = 20           // distance from trainer to client location

Expected response 200

{
  "success": true,
  "data": {
    "course_id":         2,
    "course_title":      "Mobile Coaching — I Come to You",
    "km":                20,
    "price_per_km":      5,
    "travel_cost":       100,   // 20 × 5
    "session_price":     200,   // course original_price
    "total_with_travel": 300    // 200 + 100
  }
}

1
POST AUTH (Trainer)
Trainer: Start Session
/api/trainer/sessions/{{booking_a_id}}/start
Use {{trainer_token}}. Only works when payment_status=paid. No body needed.

confirmed → in_progress

2
POST AUTH (Trainer)
Trainer: Complete Session
/api/trainer/sessions/{{booking_a_id}}/complete
Use {{trainer_token}}. No body needed.

in_progress → awaiting_confirmation

3
POST AUTH (Client)
Client: Confirm Completion
/api/trainer/bookings/{{booking_a_id}}/confirm-completion
Switch back to {{token}}. No body needed.

awaiting_confirmation → completed. can_review becomes true.

}
Only once per booking. Only after status=completed.
